# v0.10.8 — Welcome Release Awareness

**Date:** 2026-05-25

## Highlights

- Welcome now refreshes fresh `up_to_date` update-cache entries on the stable
  channel.
- Newly published stable releases can appear when opening Mirror even inside the
  normal cache TTL.
- The fix preserves the existing remote-check disable switch:
  `MIRROR_WELCOME_REMOTE_UPDATE_CHECK=off`.

## Why This Patch Exists

Production validation showed a gap between the update command and the welcome
experience. `runtime update --check` correctly detected a new stable release,
but opening Mirror did not show that a new version was available.

The cause was the welcome cache. A cached `up_to_date` result was trusted for six
hours. If stable advanced during that window, users could open Mirror and still
see no update prompt.

## What Changed

The welcome update-awareness cache now refreshes on stable when the cached state
is either `update_available` or `up_to_date`. This keeps startup awareness aligned
with newly published stable releases without asking users to run `git fetch` or
delete cache files.

## Conscious Exclusions

This patch does not remove caching and does not make non-stable channels perform
the same remote awareness refresh. The behavior is targeted at the user-facing
stable release channel.

## Next Horizon

A follow-up stub release can validate the behavior end-to-end: update production
to this version, publish a later stable tag, then open Mirror and confirm the
welcome detects the new release.
