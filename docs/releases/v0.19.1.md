# v0.19.1 — Web Console Maintenance CI Formatting

**Date:** 2026-06-02

## Highlights

- Preserves the v0.19.0 web-console journey creation and conversation deletion behavior.
- Applies repository formatting to the touched Python files.
- Restores the release line to green CI after the v0.19.0 main push failed at `ruff format --check`.

## Where We Started

The v0.19.0 release was promoted and pushed with the intended product behavior,
but GitHub Actions failed on the formatting gate. Local validation had run ruff
linting, full tests, JavaScript syntax checks, and release-note checks, but did
not include the repository-wide `ruff format --check src/ tests/` gate that CI
runs.

## What Changed

This patch applies ruff formatting to the files reported by CI and bumps the
release line to `v0.19.1`. It does not change the web-console product surface:
journey creation, single conversation deletion, selected bulk deletion, memory
preservation on delete, and broader selected-journey conversation visibility are
unchanged from v0.19.0.

## Conscious Exclusions

- No additional journey-creation behavior is added.
- No pagination or Load more behavior is added.
- No destructive-action semantics are changed.

## Next Horizon

Keep `ruff format --check src/ tests/` in the local release validation sequence
before future promotion so formatting patches do not need to follow immediately
after feature releases.
