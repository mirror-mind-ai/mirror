# v0.10.1 — Welcome Startup Clarity

**Date:** 2026-05-25

## Highlights

- Pi now shows an immediate startup status while checking sessions and memories.
- Pi shows a second status while checking release availability.
- The welcome now says `New Version Available` instead of `Update available`.
- Pi renders welcomes with a new-version notice as warning-level notifications
  for stronger visual emphasis.

## Where We Started

After publishing v0.10.0, production validation showed that opening Mirror could
take long enough for the Pi startup counter to become visible. The user could see
time passing, but not what Mirror was doing. At the same time, the release notice
was technically correct but too quiet: `Update available` sounded operational,
while the real product signal was that a new Mirror Mind version existed.

## What Changed

The Pi runtime extension now sets a status message before the startup work begins
and updates that status before release awareness is checked. This gives the user
a clear explanation while the counter is running: Mirror is checking sessions,
memories, and release status.

The welcome wording now uses `New Version Available`. When that line appears,
Pi sends the welcome as a warning-level notification instead of a normal info
notification, giving the release notice stronger emphasis within Pi's available
UI controls.

The welcome product spec was updated to reflect the new wording and emphasis
expectation.

## Conscious Exclusions

This patch does not change the self-update executor, release detection logic, or
stable-channel promotion flow.

It also does not hard-code colors. Pi owns notification rendering; Mirror can
choose the notification level, not exact text color.

## What We Learned

A release notice is not only information. It is an invitation to act. The wording
and visual weight should match the user's mental model: they are not being told
that an abstract update exists, they are being told that a new version of their
Mirror is available.

## Next Horizon

If startup remains slow in production, the next improvement should measure which
startup stage dominates: stale conversation closure, Pi session backfill,
pending extraction, release check, or welcome composition. This patch makes the
current wait legible; it does not yet optimize the wait itself.
