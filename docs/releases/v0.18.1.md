# v0.18.1 — Empty Conversation Cleanup

**Date:** 2026-06-01

## Highlights

- Expands Orphan conversation cleanup to include empty conversations.
- Adds an explicit `empty_conversations` cleanup source.
- Makes the default latest-backfill no-change cleanup include records that are orphaned or empty.
- Shows cleanup reason and journey in candidate result lists.
- Keeps deletion protected by dry-run, approval, and backup.

## Where We Started

After v0.18.0 shipped, production cleanup showed a second low-substance record
shape: conversations with `message_count = 0` but a valid journey association.
They are not orphan conversations, so the original cleanup operation correctly
left them alone even though they were just abandoned session shells.

## What Changed

The existing cleanup operation now treats low-substance cleanup as a superset of
orphan cleanup. `all_orphans` still means records with no journey. The new
`empty_conversations` source targets records with zero messages even if they have
a journey. The default `no_change_latest_backfill` source now includes latest
metadata-backfill no-change records when they are either orphaned or empty.

Candidate rows now include a cleanup reason (`orphan`, `empty`, or
`orphan_and_empty`) and the journey value, so the user can understand why each
record is eligible before approving deletion.

## Conscious Exclusions

- Non-empty conversations with a journey are still not cleanup candidates.
- Empty cleanup does not infer whether an abandoned session was important; it only exposes a preview and requires explicit approval.
- The operation remains backup-gated before deletion.

## Next Horizon

If more low-substance shapes appear, cleanup should become a clearer
Conversation Cleanup operation with named policies rather than continuing to
stretch the orphan-specific title.
